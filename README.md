Project overlook event management
It's mainly targeted for structured event management for schools which are dependent on third party software developers who are unbothered about updating it.
Hence I tried my best to create this command line interface based menu driver event manager.
It has basic data persistence through storing data in csv files.
Includes users based on roles and respective authorization and authentication (password only).

Instructions to run this:
1. Clone the whole repo onto your local system.
2. Delete EventManager.py since it's not needed
3. Update root variable to the file location where the file Account_Signin.xlsx was installed (this is like 1430 or 1428)
4. Run the code (If you've cloned the whole repo, certain notices would already be available)

Don't bother me on updating it with OOPS and DBs via ORMS and Frameworks
